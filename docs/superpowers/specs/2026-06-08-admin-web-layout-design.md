# 中文 Web 后台布局设计

## 1. 设计目标

Web 后台是内部运营工具，目标是让运营人员高效完成站点配置、小说导入、任务观察、术语维护、发布控制和合规处理。

后台布局应遵循：

- 信息密度适中，优先支持扫描、筛选和批量操作。
- 不使用营销式大图、英雄区或装饰卡片。
- 页面结构稳定，列表、筛选、详情和操作区保持一致。
- 高风险操作必须明确提示，并记录审计日志。
- 所有界面文案默认中文。

## 2. 全局布局

后台采用左侧导航 + 顶部工具条 + 主内容区结构。

```text
┌──────────────────────────────────────────────────────────┐
│ 顶部工具条：当前页面标题 / 环境标识 / 管理员 / 退出登录       │
├──────────────┬───────────────────────────────────────────┤
│ 左侧导航      │ 主内容区                                    │
│              │                                           │
│ 仪表盘        │ 页面标题                                    │
│ 站点管理      │ 筛选区 / 操作区                              │
│ 书籍管理      │ 表格 / 详情 / 表单                            │
│ 章节管理      │ 分页 / 状态 / 抽屉 / 弹窗                      │
│ 术语表        │                                           │
│ 任务管理      │                                           │
│ 分类推荐      │                                           │
│ 合规发布      │                                           │
│ 审计日志      │                                           │
└──────────────┴───────────────────────────────────────────┘
```

左侧导航宽度固定，主内容区自适应。主内容区最大宽度不强制居中，后台页面优先利用横向空间展示表格。

## 3. 导航结构

一级导航：

- 仪表盘
- 站点管理
- 书籍管理
- 章节管理
- 术语表
- 任务管理
- 分类推荐
- 合规发布
- 审计日志

导航规则：

- 当前页面高亮。
- 导航项使用图标 + 中文名称。
- 不在左侧导航里展示过多二级菜单。
- 与当前对象相关的二级入口放在详情页内，例如书籍详情中的章节、术语、任务。

## 4. 通用页面模式

### 列表页

列表页结构：

```text
页面标题
说明文字
筛选区
主操作按钮
数据表格
分页
```

筛选区包含：

- 关键词搜索。
- 状态筛选。
- 分类或站点筛选。
- 时间范围筛选。
- 重置按钮。
- 查询按钮。

表格要求：

- 首列展示主对象名称。
- 状态字段使用清晰标签。
- 操作列固定在右侧。
- 行内操作不超过 4 个，更多操作进入下拉菜单。
- 表格支持空状态、加载状态和错误状态。

### 详情页

详情页结构：

```text
返回入口
对象标题 + 状态标签
关键操作区
基础信息
关联数据 Tab
```

详情页 Tab 建议：

- 基础信息。
- 章节。
- 任务。
- 术语。
- 发布与合规。
- 审计记录。

### 表单页

表单布局：

- 使用单列或两列栅格。
- 长文本使用大文本框。
- 解析规则、过滤规则等复杂字段使用可折叠区块。
- 必填项清晰标记。
- 保存按钮固定在表单底部或右上角。

表单交互：

- 保存前做前端基础校验。
- 保存失败展示可读错误。
- 离开未保存表单时提示确认。

### 弹窗与抽屉

使用规则：

- 简单确认使用弹窗。
- 查看详情、编辑少量字段、查看失败原因使用右侧抽屉。
- 大型表单和复杂编辑使用独立页面，不塞进弹窗。

## 5. 仪表盘布局

仪表盘用于快速判断系统状态，不做复杂分析报表。

布局：

```text
核心指标行
任务状态区
异常待处理区
最近操作区
```

核心指标：

- 书籍总数。
- 章节总数。
- 今日抓取任务。
- 今日翻译任务。
- 失败任务。
- 异常章节。
- 待确认术语。

任务状态区：

- 抓取任务数量：运行中、失败、成功。
- 清洗任务数量：运行中、失败、成功。
- 翻译任务数量：运行中、失败、成功。

异常待处理区：

- 需要处理的异常章节。
- 失败次数较高的站点。
- 待确认术语。

最近操作区：

- 最近下架。
- 最近重翻译。
- 最近修改站点配置。
- 最近修改术语表。

## 6. 站点管理布局

### 站点列表

表格字段：

- 站点名称。
- 基础域名。
- 启用状态。
- 请求限速。
- 最大并发。
- 最近抓取时间。
- 最近失败原因。
- 操作。

行内操作：

- 编辑。
- 测试解析。
- 启用/禁用。
- 查看任务。

### 站点详情

详情区块：

- 基础配置。
- 请求配置。
- 章节列表解析规则。
- 章节正文解析规则。
- 广告过滤规则。
- 限速与并发。
- 解析测试。
- 关联书籍。

解析测试布局：

```text
测试 URL 输入
测试类型选择：章节列表 / 章节正文
执行测试按钮
解析结果预览
原始 HTML 摘要
错误信息
```

解析测试只用于验证规则，不直接创建正式章节。

### 书籍列表

筛选项：

- 关键词：书名、作者、来源 URL。
- 站点。
- 分类。
- 连载状态。
- 发布状态。
- 翻译状态。
- 是否推荐。

表格字段：

- 封面缩略图。
- 书名。
- 作者。
- 来源站点。
- 分类/标签。
- 连载状态。
- 最新章节。
- 更新时间。
- 发布状态。
- 推荐权重。
- 操作。

主操作：

- 添加书籍。

行内操作：

- 查看。
- 抓取最新。
- 重翻译。
- 下架/发布。
- 更多。

### 添加书籍

表单字段：

- 来源站点。
- 小说 URL。
- 书名。
- 作者。
- 封面 URL。
- 简介。
- 分类。
- 标签。
- 连载状态。
- 更新频率。
- 推荐权重。
- 广告开关。

表单提交后：

- 保存书籍和来源 URL。
- 可选择立即创建抓取任务。
- 返回书籍详情页。

### 书籍详情

顶部区域：

- 封面。
- 书名。
- 作者。
- 来源站点。
- 发布状态。
- 连载状态。
- 最新章节。
- 更新时间。

关键操作：

- 抓取最新章节。
- 重抓整本书。
- 重翻译整本书。
- 下架/重新发布。
- 隐藏/取消隐藏。

Tab：

- 基础信息。
- 章节。
- 术语表。
- 任务。
- 发布与广告。
- 审计记录。

## 8. 章节管理布局

章节管理可以从全局入口进入，也可以从书籍详情进入。

筛选项：

- 书籍。
- 章节标题。
- 抓取状态。
- 清洗状态。
- 翻译状态。
- 发布状态。
- 异常状态。

表格字段：

- 章节序号。
- 章节标题。
- 抓取状态。
- 清洗状态。
- 翻译状态。
- 发布状态。
- 字数。
- 更新时间。
- 操作。

行内操作：

- 查看。
- 重抓。
- 重新清洗。
- 重翻译。
- 隐藏/发布。

章节详情使用左右结构：

```text
左侧：章节元信息和状态
右侧：内容预览 Tab
```

内容预览 Tab：

- 原始内容。
- 清洗正文。
- 英文译文。
- 任务记录。

内容预览要求：

- 使用等宽或阅读友好的正文区域。
- 支持复制。
- 长内容需要滚动，不撑爆页面。
- 异常检测结果显示在正文上方。

## 9. 术语表布局

术语表可以按书籍过滤。

筛选项：

- 书籍。
- 术语类型。
- 启用状态。
- 中文原词。
- 英文译名。

表格字段：

- 中文原词。
- 英文译名。
- 类型。
- 说明。
- 启用状态。
- 更新时间。
- 操作。

主操作：

- 新增术语。
- 导入术语。
- 处理待确认术语。

待确认术语区：

- 中文候选词。
- 来源章节。
- 建议英文译名。
- 出现次数。
- 操作：确认、编辑后确认、忽略。

术语编辑抽屉字段：

- 中文原词。
- 英文译名。
- 类型。
- 说明。
- 启用状态。

- 任务状态。
- 目标书籍。
- 目标章节。
- 供应商。

表格字段：

- 类型。
- 状态。
- 目标对象。

- 重试次数。
- 失败原因摘要。

- 待执行。
- 执行中。
- 成功。
- 失败。
- 重试中。
- 已取消。

行内操作：

- 查看详情。

- 失败原因。

## 11. 分类与推荐布局

页面分为两个 Tab：

- 分类与标签。
- 推荐配置。

分类与标签：

- 分类列表。
- 标签列表。
- 启用状态。
- 排序权重。

推荐配置：

- 推荐位名称。
- 绑定书籍。
- 排序权重。
- 生效状态。

推荐配置不做复杂算法，只维护后台可控排序。

## 12. 合规与发布控制布局

页面分为：

- 全局合规配置。
- 版权投诉记录。
- 下架记录。

全局合规配置字段：

- 隐私政策 URL。
- 服务条款 URL。
- 广告披露文案。

版权投诉记录字段：

- 投诉来源。
- 投诉人联系方式。
- 关联书籍。
- 关联章节。
- 投诉说明。
- 处理状态。
- 处理备注。

处理状态：

- 待处理。
- 处理中。
- 已下架。
- 已驳回。
- 已关闭。

下架记录：

- 对象类型。
- 对象名称。
- 操作人。
- 操作时间。
- 原因。

## 13. 状态与颜色规则

状态标签应统一。

建议状态：

- 成功：绿色。
- 执行中：蓝色。
- 待处理：灰色。
- 警告：橙色。
- 失败/下架：红色。

不要只依赖颜色表达状态，必须同时显示文字。

## 14. 高风险操作

以下操作需要二次确认：

- 下架整本书。
- 隐藏单章。
- 删除标记。
- 取消运行中的任务。
- 重翻译整本书。
- 禁用站点。

确认弹窗需要显示：

- 操作对象。
- 操作影响。
- 是否会影响 App 展示。
- 确认按钮。
- 取消按钮。

## 15. 空状态和错误状态

空状态文案要告诉运营下一步可以做什么。

示例：

- 站点为空：`暂无站点，请先添加站点配置。`
- 书籍为空：`暂无书籍，请添加小说来源 URL。`
- 任务为空：`暂无任务。你可以从书籍详情页触发抓取或翻译。`
- 异常章节为空：`暂无异常章节。`

错误状态需要显示：

- 错误摘要。
- 可读失败原因。
- 重试入口。

## 16. 布局验收标准

布局设计进入实现前应满足：

1. 运营人员可以从导航进入所有 MVP 功能。
2. 所有列表页都有筛选、状态、操作和分页设计。
3. 高风险操作有明确确认模式。
4. 任务失败原因和异常章节有可观察入口。
5. 书籍、章节、术语、任务之间可以互相跳转。
6. 合规配置、版权投诉和下架记录有独立页面承载。
7. 页面文案默认中文，技术字段保留必要英文标识。
