# Android 阅读器布局设计

## 1. 设计目标

Android App 面向 Google Play 海外读者，MVP 默认英文界面，核心目标是让用户快速找到小说、加入书架并稳定阅读英文译文。

布局设计原则：

- 第一屏直接进入阅读产品，不做营销落地页。
- 阅读路径短：发现书籍 -> 查看详情 -> 开始阅读。
- 阅读器优先稳定、清晰和可调节。
- 不强制登录，不阻断阅读。
- 书架、最近阅读、阅读进度和设置优先本地保存。
- 广告存在但不能破坏基础阅读连续性。

## 2. 信息架构

底部导航使用 3 个主入口：

- Home
- Categories
- Bookshelf

其他页面通过页面内跳转进入：

- Book Detail
- Chapter Catalog
- Reader
- Reader Settings

结构：

```text
Home
├── Book Detail
│   ├── Chapter Catalog
│   └── Reader
Categories
├── Book List
│   └── Book Detail
Bookshelf
├── Book Detail
└── Reader
```

## 3. 全局布局规则

### 顶部栏

列表类页面使用顶部栏：

- 页面标题。
- 可选搜索入口。
- 可选刷新或更多按钮。

阅读器页面默认隐藏顶部栏，轻触正文后显示阅读控制层。

### 底部导航

底部导航仅在主入口页面显示：

- Home
- Categories
- Bookshelf

进入详情、目录和阅读器后隐藏底部导航，减少干扰。

### 加载与错误

所有远程数据页面都需要：

- 首次加载状态。
- 下拉或按钮重试。
- 空状态。
- 分页加载状态。

错误文案使用英文界面文案，示例：

- `Unable to load books. Try again.`
- `No books found.`
- `This chapter is unavailable.`

## 4. Home 首页

目标：展示可读内容和运营推荐，帮助用户进入阅读。

布局：

```text
顶部栏：HappyNovel / 搜索入口
推荐横向列表
Latest Updates
Popular
New Books
```

模块说明：

- 推荐横向列表展示封面、书名和一句短简介。
- Latest Updates 展示最近更新的书。
- Popular 展示热门书。
- New Books 展示新上架书。

书籍卡片字段：

- 封面。
- 书名。
- 作者。
- 分类标签。
- 最新章节或更新时间。

交互：

- 点击书籍进入详情页。
- 列表模块支持查看更多。
- 首页刷新时重新请求后端首页接口。

布局要求：

- 封面使用固定比例，避免列表跳动。
- 书名最多两行。
- 简介最多两行。
- 列表项高度稳定。

## 5. Categories 分类页

目标：让用户按题材、标签和连载状态筛选小说。

布局：

```text
顶部栏：Categories
分类 Tab / Chip 组
筛选行：Status / Sort
书籍列表
分页加载
```

分类入口：

- Genre。
- Tags。
- Status。

排序选项：

- Latest Update。
- Popular。
- Newest。
- Recommended。

书籍列表字段：

- 封面。
- 书名。
- 作者。
- 简介。
- 连载状态。
- 更新时间。

交互：

- 切换分类后刷新列表。
- 切换排序后回到列表顶部。
- 点击书籍进入详情。

## 6. Book Detail 书籍详情页

目标：让用户判断是否阅读，并快速进入当前章节。

布局：

```text
顶部栏：返回 / 标题 / 更多
封面 + 书籍基本信息
简介
分类标签
主操作按钮
最新章节区域
目录入口
```

基本信息：

- 封面。
- 书名。
- 作者。
- 连载状态。
- 更新时间。
- 分类标签。

主操作按钮：

- `Start Reading`
- `Add to Bookshelf` 或 `In Bookshelf`

如果本地已有阅读进度：

- 主按钮显示 `Continue Reading`
- 展示最近阅读章节。

最新章节区域：

- 最新章节标题。
- 更新时间。
- 点击进入最新章节阅读。

目录入口：

- 显示章节总数。
- 点击进入 Chapter Catalog。

广告：

- 详情页可预留轻量广告位。
- 广告不能位于主操作按钮上方。

## 7. Chapter Catalog 章节目录页

目标：让用户选择章节并查看阅读状态。

布局：

```text
顶部栏：返回 / Chapters
书名摘要
排序切换：正序 / 倒序
章节列表
```

章节列表字段：

- 章节序号。
- 章节标题。
- 本地阅读状态。
- 更新时间。

阅读状态：

- 未读。
- 已读。
- 当前阅读。

交互：

- 点击章节进入 Reader。
- 支持正序/倒序切换。
- 长列表分页加载。

## 8. Reader 阅读器

目标：提供稳定、舒适、可调节的章节阅读体验。

默认布局：

```text
正文区域
```

轻触正文后显示控制层：

```text
顶部控制层：返回 / 章节标题 / 目录
底部控制层：上一章 / 设置 / 下一章
```

正文样式：

- 段落间距稳定。
- 行距可调。
- 字体大小可调。
- 支持日间和夜间模式。
- 支持背景色切换。
- 长章节滚动阅读。

阅读控制：

- 上一章。
- 下一章。
- 打开目录。
- 打开设置。
- 保存阅读进度。

预加载：

- 进入章节后预加载下一章。
- 如果存在上一章缓存，可快速返回。

广告：

- 可支持章节间广告。
- 可支持底部广告位。
- 插屏广告只在章节切换时触发。
- 插屏频率由后端配置控制。
- 广告不能遮挡正文控制按钮。

不可用章节：

- 如果章节未发布或不可获取，显示错误页。
- 错误页提供返回目录和重试。

## 9. Reader Settings 阅读设置

设置面板从阅读器底部弹出。

布局：

```text
阅读设置
字体大小：- 当前值 +
行距：紧凑 / 标准 / 宽松
主题：Light / Sepia / Dark
背景色选项
```

设置项：

- Font Size。
- Line Height。
- Theme。
- Background。

交互：

- 调整后立即预览。
- 设置保存到 DataStore。
- 关闭面板后继续阅读。

MVP 不包含：

- 字体家族选择。
- 复杂翻页动画。
- 横竖屏高级排版。

## 10. Bookshelf 书架页

目标：让用户回到收藏书籍和最近阅读。

布局：

```text
顶部栏：Bookshelf
Recently Reading
Saved Books
```

Recently Reading 字段：

- 封面。
- 书名。
- 最近阅读章节。
- 阅读进度。
- 更新时间。

Saved Books 字段：

- 封面。
- 书名。
- 最新章节。
- 是否有更新。

交互：

- 点击最近阅读进入 Reader。
- 点击书籍进入 Book Detail。
- 支持移出书架。

空状态：

- `Your bookshelf is empty.`
- 提供返回 Home 的入口。

## 11. 缓存布局与状态

MVP 不做独立下载管理页，只在书籍详情或书架中展示简单缓存状态。

缓存策略：

- 当前阅读章节。
- 下一章。
- 上一章。
- 最近阅读书籍的少量章节。

可见状态：

- 已缓存。
- 正在缓存。
- 缓存失败。

用户操作：

- 清除某本书缓存。
- 清除全部缓存。

清除缓存入口可以放在设置或书架更多菜单中。

## 12. 本地状态与同步边界

MVP 本地保存：

- 书架。
- 阅读进度。
- 阅读设置。
- 最近阅读。
- 简单章节缓存。

后端保存：

- 匿名设备 ID。
- 阅读行为上报。
- 广告配置。

MVP 不做：

- 跨设备同步。
- Google 登录。
- 云端书架。
- 云端阅读进度。

后续接入登录后，需要支持匿名数据合并。

## 13. 视觉风格

阅读类 App 应保持安静、清晰和内容优先。

建议：

- 首页和分类页使用清晰列表和稳定封面比例。
- 阅读器背景避免高饱和颜色。
- 夜间模式需要降低对比刺眼程度。
- 按钮和图标尺寸稳定，避免点击困难。
- 卡片圆角控制在 8dp 或以内。
- 不使用装饰性渐变背景和大面积视觉噪音。

## 14. 可访问性与可读性

必须支持：

- 正文足够大的默认字号。
- 字体大小调节。
- 日夜模式。
- 足够对比度。
- 按钮可点击区域不小于常规 Android 触控建议。
- 长书名和长章节标题不挤压操作按钮。

文本溢出规则：

- 书名最多两行。
- 简介最多两行或三行。
- 章节标题在目录中最多两行。
- 阅读器章节标题可在控制层中截断。

## 15. 页面状态清单

每个远程页面至少实现：

- Loading。
- Content。
- Empty。
- Error。
- Paging Loading。

阅读器额外实现：

- Chapter Loading。
- Chapter Content。
- Chapter Unavailable。
- Previous Chapter Missing。
- Next Chapter Missing。
- Offline Cached Content。

## 16. Android 布局验收标准

布局设计进入实现前应满足：

1. 用户无需登录即可从首页进入阅读。
2. Home、Categories、Bookshelf 三个主入口清晰。
3. 书籍详情页有开始阅读、加入书架和目录入口。
4. 阅读器支持上一章、下一章、目录、设置和进度保存。
5. 阅读设置支持字体大小、行距、日夜模式和背景色。
6. 本地书架和最近阅读有明确入口。
7. 广告位置不会遮挡主阅读操作。
8. 所有远程页面都有加载、空状态和错误重试。
9. 页面结构支持后续接入 Google 登录和云同步，不需要重做主导航。
